# Setflow 기획서 — 토픽 · 에픽 · 태스크 백로그

> **문서 버전** v2.0 · 2026-07-23 (공식 PRD v2.2 정본 반영)
> **정본 기준** 개발 정본 = **공식 PRD v2.2 · IA v1.1 · Router v1.0**(작성자 김건우). React 프로토타입은 "UI 참조 구현". 정책 수치·누락 기능은 `07-공식정책-정합성보강.md` 참조.
> **목적** Setflow를 실서비스로 구축하기 위한 **개발 백로그**. 3계층 구조.
> **계층 정의**
> - **Topic (토픽)** = 제품 도메인(대분류). `T{n}`.
> - **Epic (에픽)** = 토픽 내 사용자 가치 단위 기능 묶음. `T{n}-E{m}`.
> - **Task (태스크)** = 구현 실행 단위. **[FE]/[BE]/[INFRA]/[EXT]** 태그. `T{n}-E{m}-{k}`.
> **표기** 우선순위 P0(치명)~P3(개선), 관련 페이지/라우트/화면ID 명시.
> **연관** `01-요구사항정의서.md`, `06-제품전략-경쟁분석.md`, `07-공식정책-정합성보강.md`, `03-페이지별-상세명세.md`.

> ⚠️ **v2.0 변경점**: 제품의 **유일한 차별화 코어 = "기록 마찰 제거(Zero-Friction)"** 이므로, **T12(음성·위젯·워치)** 를 **P0 최상위**로 신설. 그 외 신설: **T13 분야별 랭킹·게이미피케이션**, 정책 태스크(구독·정산·모더레이션·미성년·인증) 명시. (`06` §5 우선순위 시사점 반영)

## 개발 로드맵 개요 (Phase)

| Phase | 목표 | 포함 토픽 |
|---|---|---|
| **Phase 0 — 기반** | 인증·권한·영속성·멀티테넌시 확립 | T0, T1(부분), T11 |
| **Phase 1 — 차별화 코어 + 회원 루프** | **Zero-Friction 기록(음성/위젯/OCR)** + 운동기록·루틴·대시보드 | **T12**, T2, T3, T4 |
| **Phase 2 — 네트워크** | 커뮤니티·분야별 랭킹·루틴 소셜 | T5, **T13** |
| **Phase 3 — 코칭/수익화** | 상담·코칭·결제·정산 | T6, T7, T9 |
| **Phase 4 — B2B/생태계** | 헬스장 운영·플랫폼 관리·모더레이션 | T8, T10 |

> **뺄셈 규율(06 §5-C)**: 1차 릴리스 = **1차 타깃(직장인)의 "끊김 없는 하루" 여정 완결**에 필요한 에픽만. 트레이너 브랜딩·소속관리·랭킹은 Step 2→3로 단계 배치.
> **정량 DoD**: Zero-Friction 관련 에픽은 **Time-to-log(세트 기록 소요시간)·세트 간 조작 단계 수**를 완료 기준 지표로 포함.

# T0. 플랫폼 기반 (Foundation) — P0

> 다른 모든 토픽의 선행 조건. 프로토타입 최대 결함(전역 상태·인증 부재·영속성) 해소.

### T0-E1. 인증 · 세션 · RBAC
- **T0-E1-1** [BE] OAuth2 소셜 로그인(카카오/구글/애플) + 이메일 로그인/가입, JWT+리프레시 발급. `POST /auth/social`, `/auth/login`, `/auth/signup`. (LoginPage, AuthSignupPage)
- **T0-E1-2** [BE] `users.role` 기반 RBAC 미들웨어(general/trainer/gym/admin) + 관리자 세부 권한.
- **T0-E1-3** [FE] `RouteGuard` 서버 세션 검증 연동(`GET /me`), 역할별 홈 리다이렉트 정합화.
- **T0-E1-4** [FE] **버그픽스**: Welcome/Login에서 `setUserType` 확정, 로그인 시 `'kg',false` 하드코딩 제거 → 서버 설정 복원.
- **T0-E1-5** [BE] 게스트 익명 세션 + 가입 시 로컬 데이터 병합 `POST /auth/link-guest`.

### T0-E2. 영속성 · 상태 동기화
- **T0-E2-1** [FE] Zustand `persist` 미들웨어(localStorage) 도입, 앱 상태 복원.
- **T0-E2-2** [BE] 도메인별 REST API 스캐폴딩 + OpenAPI 스펙.
- **T0-E2-3** [FE] API 클라이언트 계층(fetch 래퍼·인증 헤더·에러·낙관적 업데이트) 도입, store를 서버 동기화형으로 리팩터.

### T0-E3. 멀티테넌시 (⚠️ 최우선 결함 G1)
- **T0-E3-1** [BE] `workout_sessions`·`routines`에 `member_id`/`owner_id` 스코프 컬럼 필수화, 전역 공유 제거.
- **T0-E3-2** [BE] 코치→회원 세션/루틴 접근을 `member_assignments`·`coaching_contracts` 권한으로 인가.
- **T0-E3-3** [FE] trainer/gym 회원 상세 화면이 `userId` 파라미터로 실제 회원 데이터 조회하도록 수정(현재 전역 store 공유).
