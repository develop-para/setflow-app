## 0. 한 줄 요약
- **Setflow의 승부수 = "기록 마찰(friction)의 완전 제거"**. 경쟁사가 *더 나은 기록장·더 똑똑한 코칭*을 만드는 동안, Setflow는 **기록이라는 행동 자체를 없앤다(음성/위젯/OCR)**.
- **Planfit** = "결정의 외주화"(AI 큐레이션으로 *무엇을 할지* 제거). **BodyCalendar** = "입력 시간의 압축"(초고속 UX로 *어떻게 빨리* 제거).
- **Setflow의 자리** = 결정 + 입력 **둘 다 제거** + 두 경쟁사 모두 약한 **소셜(루틴 커뮤니티)·B2B2C 생태계**를 얹는다.

## 1. 제품 비전 — Zero-Friction Fitness

### 1.1 Product Thesis
- 슬로건: **"운동은 끊기지 않고, 기록은 자연스럽게."** — "스마트폰 OS 생태계 + AI 기반 제로 마찰 운동 데이터 플랫폼".
- **문제 재정의**: 기존 앱의 문제는 "기능 부족"이 아니라 **기록 자체가 귀찮다**는 것. 세트 사이 짧은 휴식마다 폰을 만져야 하는 **'기록 마찰'**이 이탈 원인.
  - *Workout Friction Curve*: 60분 세션 내내 **잠금해제(Unlock)→앱 진입(Find)→타이핑(Type)** 반복 → 매 세트 집중력 스파이크.
- **포지셔닝 2×2**: X축(수동입력 High-Friction → 제로마찰 Voice&OS), Y축(개인기록 Single → 커뮤니티·플랫폼 Multi). Hevy·Fitbod·Strong 대비 **Setflow는 우상단(제로마찰 × 멀티플레이어) 단독 점유**.

### 1.2 타깃 — 순환하는 확장 생태계
| 순위 | 타깃 | 니즈 | 생태계 역할 |
|---|---|---|---|
| 1차 | **바쁜 직장인**(소비자) | 빠른 기록 툴 | 데이터 소비·실행 |
| 2차 | **중급자·헬스 유저**(크리에이터) | 루틴 생성·공유 | 콘텐츠 공급 |
| 3차 | **트레이너·헬스장**(비즈니스) | 신규 고객·전문성 노출 | B2B 세일즈·수익화 |

> 순환: 중급자가 루틴 공급 → 직장인이 소비 → 트레이너가 모인 유저에 세일즈. (개인 기록 → 모두의 자산 → 비즈니스)

### 1.3 3단계 진화 로드맵
1. **Tracker**(개인 도구): 가장 적은 행동으로 정확한 데이터 생성.
2. **Community**(네트워크): 데이터 기반 루틴 자산화·소셜 공유.
3. **Ecosystem**(마켓플레이스): 검증된 데이터·오디언스 기반 B2B 연결.

### 1.4 핵심 기능 우선순위 — 3 Core + 확장
- **Core 1. 음성 중심 UX(Voice-first)** — 최우선. "벤치 80에 5 완료" 자연어 → 구조화 데이터 자동 저장. (Apple Speech/App Intents, Android SpeechRecognizer)
- **Core 2. 초저마찰 진입(Zero-Touch)** — 앱 안 열고 잠금화면·홈 위젯에서 1-Tap Log. (WidgetKit, Live Activities)
- **Core 3. OCR 자동 변환** — 인바디 결과지·머신 화면 촬영 → OCR AI 수치 변환. (ML Kit, Apple Vision)
- **확장**: 루틴 콘텐츠 플랫폼("이번 주 직장인 인기 루틴 Top 3") / 체성분 변화 관리(HealthKit·Health Connect 양방향) / B2B2C(자격증 배지·상담 CTA).

### 1.5 기술 전략 — 네이티브 OS 온디바이스
- **별도 하드웨어 없이** 스마트폰만으로 온디바이스 제로 마찰. 입력 3종(Voice/Scan/Watch) → 온디바이스 구조화 → HealthKit·Health Connect 확산.
- 함의: **낮은 인프라 비용 + 프라이버시 + 웨어러블 비의존**.

### 1.6 디자인 전략
- **시각적 마찰 제로**: 순수 블랙 배제, 웜톤·파스텔·미니멀 화이트.
- (참고) Zero-Friction 문서의 컬러 제안: 60% Ivory `#F2F0EA` / 30% Amber `#F5A623` / 10% Semantic(Success `#2BA53E`·Error `#E84040`·Premium `#7B8FE0`).
  > ⚠️ 이는 공식 `05-디자인-컬러가이드`(Supernova `#FFCA10`)와 **다른 팔레트** — 브랜드 확정 시 정본 결정 필요.

### 1.7 BM & 성장 플라이휠
- **3기둥**: ①초저마찰 UX ②루틴 소셜 ③B2B 생태계.
- 플라이휠: 최소 행동 → 정확 데이터 → 루틴 자산·커뮤니티 → 검증 오디언스 위 B2B 수익화.
- **끊김 없는 하루**: 홈 위젯 탭으로 어제 루틴 시작 → 음성 핸즈프리 기록 → 한 달 뒤 인바디 스캔 → 성공 루틴 커뮤니티 공유.

## 2. 경쟁사 A — Planfit (결정의 외주화)
*"주머니 속 AI 트레이너 — AI 큐레이션으로 초보자의 인지적 과부하 해결."*

- **통찰**: 초보자 진입 장벽은 체력이 아니라 **'무슨 운동·몇 kg·몇 세트' 인지 과부하**. 기록(Log) 앱 → **큐레이션(Curation)** 시스템으로 진화, **'결정' 단계를 삭제**.
- **4대 축**: Brain(AI 초개인화 루틴) · Context(내 헬스장 보유 기구 기반 + **'기구 교체' 버튼**) · Guidance(폼 영상·워치 타이머) · Motivation(소셜 응원·랭킹).
- **BM**: **3억 건 데이터 해자** · **114개국·매출 40% 해외** · 한계비용 제로 IT 구조 · 유료 구독(Pro).
- **SWOT**: (S)데이터·AI·글로벌 (W)자세교정 불가·숙련자 커스텀 한계 (O)고물가 PT 대안·웨어러블 성장 (T)Hevy 추격·빅테크 건강앱·**오프라인 제휴 부재**.
- **한계**: ①AI 반복 루틴 → 숙련자 이탈 ②오프라인 기구 데이터 불일치 ③구독 허들·광고 피로.
- **로드맵**: 앱 AI → Planfit GPT·웨어러블 → 오프라인 B2B O2O.

**Setflow의 공격 지점**: Planfit 약점(숙련자 이탈·오프라인 불일치·구독 허들) → Setflow는 ①**자유 기록(음성·커스텀)** ②**OCR로 오프라인 실측 흡수** ③**B2B2C 생태계 수익**으로 대비. Planfit이 "결정"을 풀었다면 Setflow는 "기록"까지 없애 **더 하류의 마찰**을 잡는다.

## 3. 경쟁사 B — BodyCalendar (입력 시간의 압축)
*"체류 시간이 짧을수록 열광하는 역설 — 복잡함을 덜어내고 본질을 채운다."*

- **성과**: 평점 **4.9 · 50만+ 다운로드**, 입소문만으로 확보한 로열티. **마케팅 비용 제로.**
- **한 줄**: "플랜핏이 '가이드'를 팔 때, 바디캘린더는 **'기록의 쾌감'**을 판다."
- **포지셔닝**: 중·고급 × **개인 기록** = "혼자 집중하는 독고다이 Gym-Rats" 니치 점유(대중 포기).
- **철학**: 세트 종료 → **터치 3번(복사/입력)** → 자동 타이머 → 즉시 다음 세트 = **체류 시간 최소화**.
- **3대 가치**: Speed(원클릭 루틴 복사·대형 버튼·광고 없음·워치) / Insight(캘린더 히트맵·1RM·볼륨 그래프) / Focus(슈퍼세트·드랍세트·RPE·RIR·한국인 종목 DB).
- **The Moat**: 마찰없는 기록 → 극강 리텐션 → 낮은 구독 장벽 → 캐시카우(마케팅 불필요) 순환. 수익 = 광고/Pro.
- **딜레마**: ①초보 진입 장벽(AI 코칭 부재) ②비소셜 고립 ③단일 BM.
- **3 레슨**: ①UX가 곧 마케팅 ②틈새시장 쐐기 ③**기능의 뺄셈**.

**Setflow의 공격 지점**: 바디캘린더 3대 약점(초보 진입·비소셜·단일 BM)은 Setflow 3기둥(음성/OCR로 초보 쉬움·루틴 커뮤니티·B2B2C)과 **정확히 상보적**. 단, **"기록 속도" 코어 밸류는 절대 훼손 금지**(뺄셈 규율).

## 4. 3사 종합 비교

| 구분 | **Setflow(자사)** | Planfit | BodyCalendar |
|---|---|---|---|
| 한 문장 | 기록 **행동 자체 제거**(음성/위젯/OCR) | **결정**을 AI에 외주화 | **입력 시간** 극한 압축 |
| 제거 대상 | 결정 + 입력 둘 다 | 결정(무엇을) | 입력(어떻게 빨리) |
| 핵심 타깃 | 직장인→크리에이터→트레이너 | 초보자(글로벌) | 숙련 매니아(니치) |
| 대표 기능 | Voice·Widget·OCR·루틴 소셜·B2B2C | AI 루틴·기구교체·가이드·랭킹 | 원클릭복사·히트맵·RPE/RIR·워치 |
| 소셜 | ○ 루틴 커뮤니티 | △ 응원·랭킹 | ✕ 고립 |
| B2B/생태계 | ◎ 트레이너·헬스장 마켓 | △ Phase 3 계획 | ✕ |
| BM | B2B2C 생태계 + 구독 | 유료구독·데이터해자 | 광고 + Pro |
| 기술 해자 | OS 네이티브 온디바이스 | 3억 건 데이터 | UX 완성도·리텐션 |
| 최대 리스크 | 음성 정확도·**넓은 스코프** | 숙련자 이탈·오프라인 불일치 | 초보 진입·비소셜·단일 BM |

**전략 결론**: 두 경쟁사의 강점(속도·온보딩)을 흡수하고 약점(고립·구독의존·오프라인 단절)을 공략하되, 스코프가 넓은 만큼 **"기록 마찰 제거"라는 단일 코어를 흐리지 않는 우선순위 규율**이 성패를 가른다.

## 5. 기획서(토픽/에픽) 우선순위 시사점

> ⚠️ 현재 우리 `02 기획서`는 React 프로토타입 역설계 기준이라 **제품의 유일한 차별화 코어인 "Zero-Friction 기록 3종(음성·위젯·OCR)"이 앞단 토픽에 없다.** 이것이 최대 갭.

**(A) "Core 마찰 제거"를 최상위 에픽으로 승격** — 3개 문서 공통 결론 = "UX 마찰 제거 = 마케팅·리텐션".
- → **1순위 에픽 = Zero-Friction 기록 3종**(Voice UX / Home Widget·Live Activities / OCR). `07-공식정책` §1.1 EXT와 통합.
- → 완료 기준(DoD)에 **Time-to-log·세트 간 조작 단계** 정량 지표 추가(바디캘린더 프레임 차용).

**(B) 소셜·B2B는 구조적 차별점 → 축소 금지** — 두 경쟁사 모두 약한 영역.
- → 루틴 커뮤니티(팔로우+**분야별 랭킹**, `07` §1.2)와 트레이너 브랜딩/상담은 생태계 차별화 에픽으로 유지.

**(C) "기능의 뺄셈" 규율로 스코프 게이트** — Setflow는 3사 중 스코프 최대 = 최대 리스크.
- → 1차 릴리스 = **1차 타깃(직장인)의 "끊김 없는 하루" 여정 완결**에 필요한 에픽만. 트레이너 브랜딩·소속관리·랭킹은 Step 2→3으로 단계 배치.

**(D) 데이터/연동을 코어 가치와 연결** — SDK·기록조회·소속관리를 개별 기능이 아닌 **"OCR·HealthKit 온디바이스 데이터 허브"** 하위로 묶는다. 특히 OCR은 Planfit의 오프라인 약점을 직접 공략.

### 권장 우선순위 재정렬
1. **P0 (차별화 코어)**: Zero-Friction 기록 3종(Voice/Widget/OCR) + Time-to-log 지표.
2. **P1 (리텐션·1차 타깃 완결)**: 캘린더 히트맵·볼륨/체성분 인사이트, 숙련자 디테일(슈퍼세트·RPE/RIR·원클릭 복사).
3. **P2 (네트워크=Step 2)**: 루틴 팔로우/공유·인기 루틴·**분야별 랭킹**.
4. **P3 (생태계=Step 3)**: 트레이너 브랜딩·상담 리드·소속 관리·헬스장 기구 SDK.

> ※ 인용 수치(4.9·50만+·3억 건·114개국·매출 40%)는 원문 슬라이드 표기값 그대로.
