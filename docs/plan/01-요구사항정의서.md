## 1. 제품 개요 (Product Overview)

### 1.1 제품 정의
**Setflow**는 "다이어리를 꾸미듯 가볍고 즐겁게 하는 감성 피트니스 기록 앱"을 표방하는 **B2B2C 종합 피트니스 코칭 플랫폼**이다. 한 플랫폼 안에서 아래 4개 역할이 유기적으로 연결된다.

| 역할 | 영문 키(userType) | 핵심 가치 |
|---|---|---|
| 일반 회원 | `general` / `guest` | 운동 기록 · 성장 데이터 · 커뮤니티 · 코칭 구매 |
| 트레이너 | `trainer` | 회원 코칭 · 루틴 판매 · 상담 · 수익 정산 |
| 헬스장 대표 | `gym` | 소속 코치 관리 · 브랜드 루틴 · 매출/정산 |
| 플랫폼 관리자 | `admin` | 심사 · 모더레이션 · 정산 · 시스템 정책 |

### 1.2 핵심 가치 제안 (Value Proposition)
- **회원**: 세트·중량·횟수 기록이 감성 캘린더/인증샷으로 축적되고, 데이터(1RM·볼륨 추이)로 성장을 체감하며, 전문가 루틴·1:1 코칭까지 원스톱으로 소비.
- **트레이너**: 루틴을 상품화해 마켓에 노출·판매하고, 상담→코칭 전환→정산까지 수익 파이프라인을 운영.
- **헬스장**: 소속 코치·회원·브랜드 루틴·매출을 하나의 콘솔에서 통제하고 코치별 성과·수수료를 정산.
- **플랫폼**: 자격 심사·콘텐츠 모더레이션·정산·랭킹/과금 정책을 중앙에서 운영해 신뢰와 수익을 확보.

### 1.3 디자인 컨셉 (기존 PRD 계승)
- **컨셉**: 밝고 활기찬(Bright, Cute & Energetic) 피트니스 경험. 크림/오프화이트 배경 + 팝 포인트 컬러(퍼플·블루·네온 그린·소프트 오렌지).
- **형태**: 큰 고딕 폰트로 숫자·타이틀 강조, 모든 카드/버튼/모달 라운드 처리(`rounded-2xl`~`rounded-[32px]`) + 부드러운 그림자.
- **인터랙션**: 다정한 마이크로카피 + 이모지(🔥🌱🌟🛡️) + Framer Motion 탭/바텀시트 애니메이션.
- **레이아웃**: 모바일 우선(`max-w-md` 중앙 정렬 컨테이너), 하단 탭 네비게이션(`BottomNav`, 역할별 분기).

### 1.4 현재 프로토타입의 성숙도 (As-Is)
- ✅ **구현됨**: 전 페이지 UI/라우팅, 운동 세션/세트 CRUD(store), 루틴 CRUD(store), 캘린더 복사/삭제, 휴식 타이머, 1RM 계산(Epley), 트레이너 초대(store).
- ⚠️ **미구현/Mock**: 인증·세션(로그인은 검증 없이 통과), 영속성(전부 인메모리), 결제·정산·환불, 상담/코칭/커뮤니티/체성분 데이터, 관리자 전 도메인, 파일 업로드, 알림/푸시, 외부 연동(홈택스·PG·OCR·SNS).

## 2. 유저 페르소나 (Personas)

> 상위 4개 대표 페르소나 + 각 페이지의 세부 상황 페르소나(→ `03-페이지별-상세명세.md`).

### P1. 일반 회원 — "김균등" (28, 직장인)
- **배경**: 헬스 3개월 차 직장인. 주 3~4회 헬스장 방문, 운동 기록을 종이/메모앱으로 관리하다 불편함을 느낌.
- **목표(JTBD)**: ①오늘 뭘 얼마나 들었는지 빠르게 기록 ②지난 기록과 성장 추이 확인 ③막히는 부분은 전문가 루틴/코칭으로 해결 ④운동 인증으로 동기 유지.
- **행동 특성**: 무료로 시작(게스트), 만족하면 프리미엄/코칭 결제. 기록의 간편함과 시각적 성취감(캘린더 색·1RM 배지)에 민감.
- **핵심 화면**: 캘린더 → 일일 운동 → 세트 상세 / 대시보드 / 루틴 마켓 / 커뮤니티 / 코칭.
- **페인포인트**: 복잡한 입력 UI, 데이터 유실, 신뢰 안 되는 전문가 정보.

### P2. 트레이너 — "이레이나" (34, 프리랜서 코치)
- **배경**: 자격증 보유 프리랜서 PT. 오프라인 회원 외 온라인으로 코칭 상품을 팔고 싶은 1인 사업자.
- **목표(JTBD)**: ①검증 배지로 신뢰 확보 ②루틴을 상품화해 마켓 노출·판매 ②상담을 코칭 계약으로 전환 ④회원 밀착 관리(피드백 72h SLA) ⑤투명한 정산 수령.
- **행동 특성**: 홈 대시보드로 하루 시작(수익·회원·할 일), 미읽음 상담을 빠르게 처리(인기도↑), 정산·환불에 민감.
- **핵심 화면**: 트레이너 홈 / 루틴 관리·통계 / 상담·리타겟팅 / 회원 상세·코칭 캘린더 / 정산·환불 / 플랜·배지 갱신.
- **페인포인트**: 자격 갱신 누락 시 활동 제한, 부당 환불로 인한 정산 차감, 회원 데이터 접근 권한.

### P3. 헬스장 대표 — "박센터" (45, 센터 오너)
- **배경**: 회원 1,200명 규모 헬스장 대표. 코치 여러 명과 회원·매출을 엑셀로 관리하던 사업주.
- **목표(JTBD)**: ①사업자 인증 후 센터 등록 ②코치 초대·회원 배정·성과 관리 ③브랜드 루틴 승인/배포 ④매출·코치 수수료·환불 정산 ⑤요금제로 회원 한도 확장.
- **행동 특성**: 홈에서 총 회원/예상 수익 확인 → 관리 액션 분기. 코치 성과(이행률·전환율)와 정산 정확도를 중시.
- **핵심 화면**: 센터 홈 / 코치 관리·배정·성과 / 브랜드 루틴 / 상담 수신함 / 회원 관리·코칭 / 정산·코치배분·환불 / 요금제.
- **페인포인트**: 코치-회원 매칭 관리, 정산 배분 투명성, 회원 한도.

### P4. 플랫폼 운영 관리자 — "정어드민" (운영팀)
- **배경**: 플랫폼 CS/심사/재무/모더레이션/인프라를 담당하는 내부 운영자(세부 역할 분화).
- **목표(JTBD)**: ①트레이너/헬스장 자격 서류 심사(SLA) ②회원 제재·배지 관리 ③콘텐츠/신고/어뷰징 모더레이션 ④코칭 정산·환불·수수료·탈퇴 최종정산 ⑤랭킹/OCR/플랜/키워드/서버 등 시스템 정책 운영.
- **행동 특성**: 대기 큐를 SLA·심각도순으로 처리, 모든 조치는 감사로그·유저 알림을 동반해야 함.
- **핵심 화면**: 회원/심사큐/제재/배지 · 콘텐츠검토/신고/위험행동 · 정산/환불/수수료/최종정산 · 랭킹/OCR/플랜/키워드/로그.
- **페인포인트**: 권한(RBAC) 부재, 처리 이력 추적, 외부 연동(홈택스·PG·OCR) 신뢰성.

## 3. 기능 요구사항 (Functional Requirements)

> 도메인(=기획서 Topic)별 요구사항. `FR-{도메인}-{번호}`. 상세 태스크는 `02-기획서`.

### FR-AUTH — 인증 · 온보딩 · 계정
- FR-AUTH-01 앱 최초 실행 시 역할 선택(Welcome) → 역할별 온보딩 분기(회원/트레이너/헬스장).
- FR-AUTH-02 소셜 로그인(카카오/구글/애플) + 이메일 로그인/가입, JWT/세션 발급. **(현재 검증 없이 통과 → 실 OAuth 필요)**
- FR-AUTH-03 회원 온보딩 4단계: 환경설정(단위·테마) → 목표(최대 2) → 신체정보 → 가입/게스트 시작. 각 단계 "건너뛰기" 허용.
- FR-AUTH-04 게스트 모드(기기 로컬 저장) + 후속 가입 시 로컬 데이터 서버 병합(link-guest).
- FR-AUTH-05 역할 확정 시점 정의: 현재 Welcome/Login에서 `setUserType` 미호출 버그 → 온보딩 완료/로그인 시 역할 확정.
- FR-AUTH-06 로그인 성공 시 서버에서 설정(단위/테마)·프로필 복원. **(현재 `'kg',false` 하드코딩이 설정 덮어씀 → 수정)**
- FR-AUTH-07 계정 설정: 프로필/구독/담당 트레이너 조회·수정, 로그아웃, 비활성화/탈퇴.
- FR-AUTH-08 개인정보 동의 관리(체성분 공유·마케팅), 동의 이력 로깅(법적 근거).

### FR-WORKOUT — 운동 기록 엔진
- FR-WO-01 월간 캘린더(6주)에 날짜별 완료상태(전체/부분/미완료)·부위·볼륨·주간요약 시각화.
- FR-WO-02 일일 세션: 운동 추가(라이브러리)·세트 입력(무게/횟수)·완료 토글·세트 추가/삭제·운동 재정렬.
- FR-WO-03 세트 완료 시 1RM 산출(Epley 기본, 사용자 선택 공식 Brzycki/Lombardi) + 휴식 타이머 자동 시작.
- FR-WO-04 세션 복사(드래그앤드롭/컨텍스트 메뉴, 완료상태 초기화)·삭제.
- FR-WO-05 개별 운동 세트 상세: 큰 입력 UI(±조정), 이전 기록 참조(**현재 하드코딩 → 실 조회**), 세트별 휴식 프리셋.
- FR-WO-06 마스터 운동 카탈로그 검색·부위/즐겨찾기/최근 필터·다중선택 추가·커스텀 운동 생성.
- FR-WO-07 휴식 타이머 정확도: `startedAt` 기반 절대시간 복원(백그라운드/리마운트 내성).
- FR-WO-08 세트 타입(normal/warmup/drop/fail)·RIR·메모 지원(설정에 따라 노출).

### FR-ROUTINE — 루틴 & 전문가 마켓
- FR-RT-01 내 루틴 CRUD(무료 4개 한도, 서버 강제), 오늘 일정에 적용, 복사.
- FR-RT-02 루틴 빌더: 이름/설명/색상 + 운동·세트 구성 + 장비 필터(**equipment 컬럼 신설 필요**).
- FR-RT-03 전문가 루틴 마켓: 검색·태그·정렬(랭킹/최신/조회/상담/코칭/후기) + 페이지네이션(**현재 미동작**).
- FR-RT-04 마켓 상세: 구성·트레이너·후기 열람, 내 루틴 저장(import, **현재 미저장**), 상담/코칭 전환.
- FR-RT-05 트레이너 루틴 상품화: 금지 키워드 실시간 검열(서버 재검증), 난이도·목표(≤3)·해시태그(≤5), 심사 상태머신(draft→review→approved/rejected).
- FR-RT-06 루틴 마켓 통계: 조회/저장/상담전환/노출 랭킹 점수(배지 가중치).

### FR-ANALYTICS — 대시보드 & 체성분
- FR-AN-01 성장 대시보드: 운동 빈도(주/월)·1RM 추이·볼륨 추이 차트(**실 세션 집계 연동 필요, 현재 mock**).
- FR-AN-02 체성분 기록: 날짜별 체중/골격근량/체지방률/BMI 수동 입력·저장(**현재 미저장**).
- FR-AN-03 프리미엄: 결과지 OCR 자동입력·AI 추이 리포트(구독 게이트).
- FR-AN-04 체성분 트레이너 공유(동의 게이트) → 코치 회원 상세에서 열람.

### FR-COMMUNITY — 커뮤니티
- FR-CM-01 인증 게시물 그리드 피드, 정렬(최신/좋아요/댓글).
- FR-CM-02 글 작성: 사진 업로드 + 오버레이(날짜/시간/위치/완료루틴) + 외부 SNS 공유.
- FR-CM-03 게시물 상세: 좋아요·댓글(100byte 제한)·신고.
- FR-CM-04 위치(GPS 역지오코딩)·완료 루틴 자동 파생.

### FR-COACHING — 코칭 · 상담 · 결제
- FR-CO-01 코칭 홈: 진행 코칭 KPI(체지방/허리/근육/체중 진행률), 내 상담 내역.
