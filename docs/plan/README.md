## 읽는 순서
1. **06 제품전략** — 왜 만드는가(북극성·차별화·경쟁).
2. **01 요구사항정의서** — 무엇을 만드는가.
3. **04 유저저니맵** — 누가 어떻게 경험하는가.
4. **02 기획서 백로그** — 어떤 순서로 개발하는가.
5. **07 공식정책-정합성** — 정본 수치·누락 기능·정본 결정(01/02 보강 필수).
6. **03 페이지별 명세** / **05 디자인가이드** — 구현 참조.

## 핵심 요약
- **역할 5종**: 일반회원 / 트레이너 / 헬스장 대표 / 플랫폼 관리자 (+게스트).
- **제품 차별화**: "기록 마찰 제거"(음성·위젯·OCR) + 루틴 소셜 + B2B2C 생태계.
- **현재 상태**: 프로토타입은 UI·라우팅 완성, 그러나 **인메모리 mock**(인증·영속성·결제·정산·모더레이션 미구현).
- **최우선 결함(P0)**: ①세션/루틴 전역 공유(member_id 스코프 부재) ②인증·RBAC 부재 ③영속성 전무.
- **⚠️ 정본 결정 필요**: 01~03(React 역설계) vs `share/` 공식 정본(PRD v2.2 등)의 화면·라우트·정책 차이 → `07` §0 참조. 권장: **공식 정본을 개발 정본으로 채택**.
