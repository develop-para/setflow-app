## TL;DR
- **브랜드 메인**: `#FFCA10` "Supernova" 옐로우 (RGB 255,202,16 / HSL 47°,100%,53%).
- **보조 중립**: `#241F20` "Thunder"(거의 블랙), `#E3E5E5` "Iron"(쿨 라이트 그레이).
- **테마**: 흰 배경 + 어두운 텍스트 + 옐로우 액션 액센트의 **라이트 테마 단일 운영**이 기본. **다크 모드는 향후 로드맵, 초기 미지원.**
- **포지셔닝**: 에너지(고채도 옐로우) + 신뢰(다크 블랙 중립) + 깔끔함(화이트 베이스).

> ⚠️ **주의**: 이 공식 가이드는 "라이트 단일 + 옐로우 브랜드"다. React 프로토타입/기존 PRD의 "크림·팝 컬러(퍼플·블루·네온그린)" 서술과 방향이 다르므로, **브랜드 확정 시 어느 쪽을 정본으로 할지 결정 필요**.

## 1. 브랜드 팔레트 (Primary 3색)
| Role | HEX | Name | RGB | HSL | CMYK |
|---|---|---|---|---|---|
| Primary(메인) | `FFCA10` | Supernova | 255,202,16 | 47°,100%,53% | 0,21,94,0 |
| Secondary(다크 중립) | `241F20` | Thunder | 36,31,32 | 348°,7%,13% | 0,14,11,86 |
| Tertiary(라이트 중립) | `E3E5E5` | Iron | 227,229,229 | 180°,4%,89% | 1,0,0,10 |

**용도**: 옐로우 = 주요 액션 버튼·아이콘·액센트 / Thunder = 본문 텍스트·워드마크 / Iron = 카드 구분선·디바이더.

## 2. UI 컬러 시스템 (라이트 모드)

**배경(Background)**
| Token | HEX | 비고 |
|---|---|---|
| App Surface(메인) | `#FFFFFF` | 대시보드·카드 배경 |
| Top Bar | `#F4F6F9` | 상단 바 |
| Card Border/Divider | `#E3E5E5` | Iron |
| Secondary Surface | `#F7F8FA` | 그룹·리스트 배경 |

**텍스트(Text)**
| Role | HEX | 비고 |
|---|---|---|
| Primary | `#241F20`(Thunder) | 본문·헤더 |
| Secondary | `#6B7280` | 메타·라벨 |
| Disabled/Hint | `#9CA3AF` | 플레이스홀더 |
| Inverse | `#FFFFFF` | 옐로우·다크 배경 위 |

**버튼(Buttons)**
| Type | Background | Text |
|---|---|---|
| Primary CTA | `#FFCA10` | `#241F20`/`#000000` |
| Secondary | `#FFFFFF` + `#241F20` 보더 | `#241F20` |
| Tertiary/Text | Transparent | `#241F20` |
| Disabled | `#E3E5E5` | `#9CA3AF` |

**아이콘 / 구분선**
- Top Bar 아이콘: 라이트 톱바 블랙 `#241F20`, 다크 배경 위 화이트 `#FFFFFF`.
- Bottom Nav: 비활성 그레이 + 활성 다크. 옐로우 액센트는 **핵심 액션·뱃지에만 한정**.
- Divider: `#E3E5E5`(Iron) 또는 더 옅은 `#EEF0F2`.

## 4. 상태(State) 색상
| State | HEX | 적용 |
|---|---|---|
| Success | `#22C55E` | 완료·성공 체크 |
| Warning | `#F59E0B`(앰버) | 주의 — **브랜드 옐로우와 명확히 구분** |
| Error/Destructive | `#EF4444` | 오류·삭제·위험 |
| Info/Neutral | `#3B82F6` | 안내 토스트 |

> ⚠️ Warning은 반드시 앰버(`#F59E0B`)/오렌지(`#FF9500`). 브랜드 옐로우 `#FFCA10`을 Warning에 쓰면 액센트와 충돌.

## 5. Gradient 정책
- **원칙**: 솔리드(단색). **공식 그라디언트 시스템은 정의하지 않음.**
- **허용 예외**: 마케팅·온보딩의 옐로우→옐로우오렌지 미세 톤, 이미지 위 다크 오버레이. **단, 시스템 토큰으로 관리하지 않음.**

## 6. 컬러 토큰 정의

**1단계 — 코어 토큰 (확정, 변경 금지)**
```css
--brand-primary: #FFCA10;      /* Supernova */
--brand-on-primary: #241F20;   /* 옐로우 위 텍스트 */
--text-primary: #241F20;
--surface-bg: #FFFFFF;
--surface-elevated: #F4F6F9;
--divider: #E3E5E5;            /* Iron */
```

**2단계 — 보조 토큰 (구현 후 컬러 픽커 재측정)**
```css
--text-secondary: #6B7280;
--text-disabled: #9CA3AF;
--state-success: #22C55E;
--state-warning: #F59E0B;    /* 앰버 */
--state-error: #EF4444;
--state-info: #3B82F6;
```

**코드베이스 연동(Flutter 대응표)** — Flutter 포팅 시 `app_colors.dart` 상수/`ColorScheme` 매핑:
| CSS 변수 | Dart 상수(예) | ColorScheme |
|---|---|---|
| `--brand-primary` | `AppColors.brandPrimary = Color(0xFFFFCA10)` | `primary` |
| `--brand-on-primary` | `Color(0xFF241F20)` | `onPrimary` |
| `--text-primary` | `Color(0xFF241F20)` | `onSurface` |
| `--surface-bg` | `Color(0xFFFFFFFF)` | `surface` |
| `--divider` | `Color(0xFFE3E5E5)` | `outlineVariant` |

## 7. 접근성 (WCAG)
| 조합 | 대비 | 판정 |
|---|---|---|
| `#FFCA10` 위 `#241F20` | ~11.7:1 | AAA ✅ |
| `#FFCA10` 위 `#FFFFFF` | ~1.5:1 | ❌ 불통과 |
| `#E3E5E5` 위 `#241F20` | ~14:1 | AAA ✅ |
| `#6B7280` 위 `#FFFFFF` | ~4.7:1 | AA ✅ |

> **핵심 원칙**: 옐로우 배경 위에는 **절대 흰색 텍스트 금지** — 반드시 `#241F20`(Thunder) 또는 `#000000`. 보조 텍스트는 `#6B7280`보다 옅게 설정하지 않는다.

## 8. 브랜드 무드·전략
- 경쟁 서비스가 차분한 한색(블루·그린·보라)을 쓰는 것과 달리, Setflow는 **에너지·동기부여**를 전면에.
- **감성 키워드**: Motivation · Energy · Progress · Approachability.
- **조합 공식**: 에너지(고채도 옐로우) + 자신감(다크 블랙 중립) + 가독성(화이트 베이스).

## 9. 리스크 & 업데이트 트리거
- **리스크**: ①Warning 옐로우 중복 → 앰버 분리 ②텍스트 위계를 그레이 명도만으로 표현 시 WCAG AA 미달 가능.
- **업데이트 트리거**: 브랜드 아이덴티티 공식 변경 / 메이저 디자인 시스템 개편 / **다크 모드 공식 지원 결정** / 보조 토큰 실측값 확정 시.
