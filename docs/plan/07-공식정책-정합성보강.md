## 1. 우리 문서에 누락된 기능 (공식 정본엔 존재)

### 1.1 스마트워치 & 위젯 (EXT 영역) — Must Have, 우리 문서 전무
- **EXT-01 잠금화면 위젯**: 세트 완료 1탭(FR-033/FR-036).
- **EXT-02 홈화면 위젯**: 소형/중형 출석 7칸 주간 요약.
- **EXT-03 스마트워치 앱**: 세트 확인/완료 동기화·타이머 제어·오프라인 로컬 저장(FR-032/FR-035).
- → **기획서 신규 토픽 필요**: `T12. 워치·위젯(EXT)`. (TBD-05 워치 개발 순서 미확정)

### 1.2 분야별 랭킹 시스템 — 초기 기획 핵심, 우리 문서 전무
- 운동 목적별 카테고리 랭킹: **슬림/다이어트 · 보디빌딩 · 러너 · 근력강화 · 체형교정 · 코어강화 · 스트레칭** × 강도(**하드코어/노멀/라이트**).
- 동일 목표 유저 간 기록 비교·경쟁. (랭킹 산식은 §3.3)
- → `T3/T4`에 **분야별 랭킹 에픽** 추가, 루틴/기록에 `category`·`intensity` 속성 필요.

### 1.3 배지 & 게이미피케이션 (DB-03) — 우리 문서엔 라우트/화면 없음
- 대시보드 하위 독립 섹션. 연속사용/루틴정복/체지방감량/목표달성/커뮤니티 배지.

### 1.4 SDK·외부 연동 확장 — 우리 §7에 일부만
- **웨어러블 기기 · 헬스장 장비 · 체성분 측정기(InBody 등) · 외부 운동 앱** 데이터 연동 SDK 검토(기록 자동화).

### 1.5 누락 화면·라우트
| 공식 경로/화면 | 내용 | 우리 상태 |
|---|---|---|
| `/auth/forgot-password` | 비밀번호 찾기 | 없음 |
| `/subscribe/upgrade` | 프리미엄 결제 전용 화면 | 팝업만 |
| `/subscribe/pause` | **구독 일시정지**(1~3개월·연 2회) | 없음 |
| `/onboarding/guide` (OB-04) | 앱 사용 가이드 슬라이드 | 없음 |
| `/dashboard/badges` (DB-03) | 배지&게이미피케이션 | 없음 |
| `/trainer|admin/settlement/commission` | 수수료 등급 | 부분/누락 |

### 1.6 입력·기록 상세 (FR-030~037)
- **입력 타입 4종**: 무게×횟수 / 시간(MM:SS) / 거리(km·m) / 무게+시간.
- **운동 라이브러리 600종**(iOS 600·Android 500) 기초 데이터셋 확보 계획.
- 커스텀 운동 생성, RIR 선택입력(기본 OFF), 캘린더 셀 표시 항목 사용자 변경.
- 세트 입력 상세: 비교박스(이전기록), ±5/±1, 세트종류 토글(일반/웜업W/드랍D/실패F), Undo 5초.
- 세트 완료 시 **3병렬 처리**: 타임스탬프 <100ms / 타이머 시작 / 1RM·볼륨 <200ms.
- 로컬 우선 저장·오프라인(동기화 충돌 = 타임스탬프 병합).

## 2. 화면 ID 체계 (IA v1.1) — 개발 추적 기준

공식 화면 ID를 우리 페이지/향후 Flutter 위젯과 매핑해야 FSD·추적이 가능.

**일반 유저**: `OB`(온보딩) · `CAL`(캘린더/홈) · `RT`(루틴) · `DB`(대시보드) · `CM`(커뮤니티) · `CS`(코칭&상담) · `SET`(설정) · `EXT`(위젯&워치)
- CAL-01 메인캘린더 · CAL-02 일일기록 · CAL-03 운동 라이브러리 · CAL-04 세트입력
- RT-01 내 루틴 · RT-02 전문가 마켓
- DB-01 통계 · DB-02 체성분(OCR) · DB-03 배지
- CS-01 상담서식 · CS-02 상담내역 · CS-03 코칭결제 · CS-04 피드백 박스(72h 환불)
- SET-01 계정&프로필 · SET-02 운동기록 환경설정 · SET-03 알림 · SET-04 데이터&개인정보 · SET-05 디스플레이
- EXT-01 잠금화면 위젯 · EXT-02 홈화면 위젯 · EXT-03 워치

**트레이너 `T-`**: T-OB · T-HOME · T-RT · T-CS · T-MEM · T-PAY · T-SET (플랜별 접근)
**헬스장 `G-`**: G-OB · G-HOME · G-TR(엔터전용) · G-RT · G-CS · G-MEM · G-PAY · G-SET
**어드민 `ADM-`**: ADM-USR · ADM-CMS · ADM-PAY · ADM-SYS (웹 어드민 `admin.setflow.app` 분리 권장)

> **네이밍 차이 주의**: 운동 라이브러리 = 공식 `/calendar/library` ↔ React `/library/:date`. 정산 세부 = 공식은 `commission`+`refunds` 분리, React는 `refunds`만.
> **스코프 초과 주의**: React 전용 화면(`/trainer|gym/workspace`, 회원 캘린더 드릴다운 `/{trainer,gym}/members/:id/calendar...`)은 공식 IA/Router엔 없음 → "React 확장" 또는 "스코프 밖"으로 결정 필요.

## 3. 정책 수치 (PRD v2.2) — 우리 문서에 없던 확정값

### 3.1 구독·결제 (PRD 8장)
- **통화 = USD($)**. 무료 / 프리미엄 **$3.99월 · $29.99년**.
- **트레이너/헬스장 플랜**: 스타터(무료, 수수료 30%) / 프로 / 엔터프라이즈 — 구간별 **$9.99~$29.99**.
- **구독 일시정지**: 최소 1개월·최대 3개월·**연 2회**.
- **다운그레이드**: 데이터 **삭제 금지 → 잠금(열람 가능)** 처리, 30일 유예.
- **결제 실패 단계**: D+0 ~ D+30 단계적 처리.
- **연간 중도 해지 환불 공식**: `연 결제액 − (월 정가 × 사용 개월수)`.
- (미확정 TBD-16 연간 할인율)

### 3.2 수익 분배·정산 (PRD 13장)
- **수수료 등급 70/75/80%**(인증 배지 → 75%, 상위 → 80%).
- 코칭 가격 범위 **$1.99~$99.99**.
- **에스크로 정산**: 단기 = **D+7 전액**, 장기 = **월 분할**. 정산 확정 = 매월 1일, 환불 7일 후.
- **72시간 피드백 보장제**: 미이행 시 **랭킹 -100점 패널티**.

### 3.3 랭킹 알고리즘 (PRD 12.4) — 분야별 랭킹의 산식
- `Final_Score = Base_Score / (T+2)^G` (HN 스타일 시간 감쇠).
- **Base_Score 가중치**: 조회 V=1 · 댓글 C=5 · 저장 S=20 · 결제 P=50 · 패널티 E=−100.
- **Time Decay G = 1.5**.
- **어뷰징 방지**: 조회 24h 1회·5초 이상 · 댓글 10자 이상 · 신규 계정 0.5배.
- → 관리자 랭킹 설정 화면(AdminSystemRanking)과 연동.

### 3.4 콘텐츠 정책·제재 (PRD 12.1~12.3)
- **위반 3등급 + SLA**: Red 1h / Orange 24h / Yellow 72h.
- **헬스 특화 금지**: 저체중 미화 · 비현실 다이어트 · 불법 보조제 · 질환자 처방.
- **제재 4단계**: 경고 → 7일 정지 → 30일 정지+**랭킹 초기화** → 영구 정지. (트레이너는 1단계 강화)
- 신고 7분류 + 누적 자동 블라인드.

### 3.5 개인정보·데이터 접근 (PRD 11장)
- 체성분 = 민감정보. **AES-256 / TLS / 열람 이력** 기록.
- **트레이너 접근 범위표**: 구매일 이전 1개월~ 등 범위 제한.
- **동의 4종**: 코칭(필수) / 체성분(선택·기본 OFF) / 피드백(ON) / 마케팅(OFF).
- 게시물 워터마크: ID+일시.

### 3.6 탈퇴·계정 (PRD 14장)
- 유저/트레이너 탈퇴 플로우, **30일 유예**, **재가입 정책**(강제 탈퇴 시 3년 금지).

### 3.7 미성년자 보호·정신건강 (PRD 15장)
- **연령 3구간**: 만 14 미만 불가 / 14~17 제한(커뮤니티 팔로워 공개만·코칭 법정대리인 동의·트레이너 DM 그루밍 방지) / 18+ 전체.
- **위험 신호 감지**: 자해 표현 → **1393(자살예방)·1388(청소년)** 연결.

### 3.8 트레이너 인증 (PRD 16장)
- 자격증 **2종 이상**(국가공인·민간·신분증 조합) + 사업자.
- **심사 SLA 3영업일**(내부 심사), 거절 사유 명시.
- **배지 연 1회 갱신**. 허위 서류 = 즉시 정지 + **3년 금지**.

### 3.9 비기능(NFR, PRD 9장)
- 콜드스타트 3초 · 저장 500ms · 검색 1초 · OCR 5초.
- GDPR · AES-256 · 업타임 99.5% · iOS16+/Android12+.
- 운동기록 전용: 터치타깃 44pt · 광고 금지.

### 3.10 계산식·데이터 (PRD 4·10장, 부록)
- **1RM = Epley 기본**(Brzycki·Lombardi 참고). 볼륨 = Σ(weight×reps).
- **논리 엔티티 4종**: workout_session / workout_exercise / workout_set / exercise_master. → ERD 정식 작성 필요(PRD 부록 B에서 미작성 명시).
- **OCR 지원**: InBody 270/370/570/770 필수, 성공률 ≥92% · 정확도 ≥98%. 월 20건 무료.

## 4. 용어 정리 (PRD 2장 Glossary — 우리 문서와 용어 불일치 주의)
- `workout_program`(개별 운동) ≠ `routine`(연속 흐름). ≠ `expert_routine`(브로셔 = 직접 판매 아님, 코칭 유입 통로).
- `inquiry`(비동기 서면 상담) · `coaching_purchase`(결제 후 담당 지정) · `completion_rate` · `ranking_score` · `escrow_settlement`.
- `set_type` = 일반/웜업(W)/드랍(D)/실패(F). `estimated_1RM`(Epley). `RIR`.
